CSS text-wrap: balance;

  1. Unpredictable line breaks ? Let the browser handle it for you.


  2. What problem does it solve ?

    Developers need not worry about the line breaks anymore. text-wrap: balance draws its roots from the classic typography technique of hand-authoring line breaks for balanced text blocks. This property enables the browser to automatically balance the wrapped text once the text-wrap: balance property is set on it or the parent div without affecting the element's width. The only catch is that it works only for six or fewer lines, as the text needs to wrap and, therefore, should have a maximum line length applied from somewhere. It is inheritable. It was mainly created for headlines, block quotes, and other display texts.


  3. How does it solve this problem ?

    The browser effectively performs a binary search for the smallest width , stopping at one CSS pixel. This does not create any additional lines. This is further optimized as the browser starts with 80% of the average line width.

  4. At what cost ?

    The browser needs to loop over iterations for getting the best balanced result. This overhead is kept under control by the six lines and under rule i.e., the browser implements this property only if there are less than 6 lines. It is generally recommended as a best practice to use this property for 4 lines or fewer to prevent the slow rendering of the page.

  5. Things to look out for :

    Interactions with the white-space property This behavior of text-wrap:balance may interfere with other properties like white-space whose motto is to prevent wrapping. That is the reason we need to unset the white-space property so that our results are as desired.
